# 本轮前端改进 — 验收操作指南

> 共 11 项改动，1317 个后端测试全部通过。按以下顺序逐一验收。

---

## 0. 启动环境

```bash
# 终端 1：后端
cd research-harness-oss
python -m research_harness_mcp.http_server

# 终端 2：前端
cd research-harness-oss/web
npm run dev
```

打开浏览器 http://localhost:3000

---

## 1. 导航栏命名（4 字中文）

**看什么：** 左侧导航栏，每项应为 4 字：今日概览、研究中心、文献资料、学术发现、研究汇报、模型配置、用量统计、系统设置

**操作：** 切换语言到中文（如果不是），目视检查即可。

---

## 2. 系统设置页 i18n

**看什么：** 进入「系统设置」页面，所有文字应为中文，不再有硬编码英文。

**操作：** 点击左侧「系统设置」，确认标题、副标题、三个卡片（评分规则、趋势管线、模型与 API 密钥）均为中文。

---

## 3. 学术发现页 i18n

**看什么：** 进入「学术发现」，Hero 区域和各 section 标题应为中文。

**操作：** 
1. 点击左侧「学术发现」
2. 确认：「学术领域在关注什么？」（hero 标题）
3. 确认：「正在升温」「值得投入的开放问题」「值得探索的方向」等 section 标题
4. 如果没有数据，会看到中文空状态提示

---

## 4. 模型配置页 — 提供商检测 + 分层路由

**看什么：** 「模型配置」页面应显示三个区域：已注册智能体、已检测的提供商、分层路由

**操作：**
1. 点击左侧「模型配置」
2. 确认「已检测的提供商」区域显示你环境变量中配置的 provider（如 Anthropic、OpenAI 等）
3. 确认「分层路由」显示 轻量 / 中等 / 重度 三张卡

---

## 5. 文献资料页 — 引用量 + 精读标签 + 相关度条件显示

**看什么：** 文献表格新增「引用量」列，标题旁有精读标签，全局视图不显示「相关度」列

**操作：**
1. 点击左侧「文献资料」
2. 确认表头有「引用量」列（带星号图标）
3. 确认**没有**「相关度」列（全局视图下）
4. 如果有论文已标为精读，标题旁会显示橙色「精读」badge
5. 在 topic 下筛选时，「相关度」列应重新出现

---

## 6. 用量统计页 — 图表可视化

**看什么：** 预算页面应有饼图（按模型）、柱状图（按阶段）、面积图（每日趋势）

**操作：**
1. 点击左侧「用量统计」
2. 如果有使用数据，确认看到三种图表而非纯 CSS 色条
3. 如果没有数据，会显示「本月暂无使用记录」

---

## 7. 研究汇报 — PDF 下载

**看什么：** 汇报列表中「下载 PDF」按钮可用

**操作：**
1. 点击左侧「研究汇报」
2. 如果有已生成的汇报，点击「下载 PDF」
3. 应弹出浏览器打印对话框（Ctrl/Cmd+P 样式），选「保存为 PDF」
4. 打印样式：隐藏导航、A4 页面、衬线字体

---

## 8. 研究中心 — 整理未分类 topic（新建 domain）

**看什么：** 如果有未分类的 topic，点「分类整理」进入 reconcile 页面

**操作：**
1. 点击左侧「研究中心」→「分类整理」
2. 在某个 topic 的 domain 下拉中，选择最后一项「+ 新建 domain...」
3. 应出现内联输入框，可能有 AI 推荐的名称
4. 输入名称后点「创建并归入」
5. 如果所有 topic 都已归类，会看到「所有 topic 都已归类」提示

---

## 9. Bug 修复：重复 React key

**看什么：** 打开浏览器 Console（F12），进入任意 topic 详情页，不应再有 `duplicate key` 警告

**操作：** 进入一个 topic 详情页，打开 DevTools Console，确认没有 key 相关 warning。

---

## 10. Bug 修复：推进阶段 "No orchestrator run found"

**看什么：** 在 topic 详情页点「推进到 XXX」不再报错

**操作：**
1. 进入一个 topic 详情页
2. 点击 hero 区域的「推进到 XXX」按钮
3. 应正常推进（如果之前没有 orchestrator run，会自动创建）

---

## 11. 精读标记功能

**看什么：** 可以把论文标为「精读」或取消

**操作：**
1. 在文献资料页，找到一篇论文
2. 右键或点击操作按钮，选择「标为精读」
3. 标题旁应出现橙色「精读」badge
4. 再次操作可「取消精读」

---

## 12. RAG Demo 脚本

```bash
cd research-harness-oss

# 先预览（不执行任何操作）
./scripts/demo_rag_topic.sh --dry-run

# 仅入库论文，不消耗 LLM token
./scripts/demo_rag_topic.sh --no-llm

# 完整流程（需要配置 LLM provider）
./scripts/demo_rag_topic.sh
```

脚本会：创建 NLP 领域 + RAG-Survey-2024 topic → 入库 15 篇 RAG 经典论文 → 元数据增强 → 下载 PDF → 分析（claim / gap / baseline）→ 生成大纲和草稿 → 初始化 orchestrator → 刷新趋势。
