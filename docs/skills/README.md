# Research Harness Skills

This directory documents the **skill subsystem** — how Research Harness ships
reusable Skill packages and how any agent platform can consume them without
RH having to know about the agent in advance.

## What is a skill?

A skill is a folder under [`skills/`](../../skills/) that contains a
`SKILL.md` file with a YAML front-matter header (the format used by Claude
Code and Codex). The body is free-form Markdown — instructions, reference
material, templates — that an agent reads when the user's intent matches the
skill's natural-language triggers.

Skills are vendor-neutral. The same `SKILL.md` is consumed by Claude Code,
Codex, OpenClaw, and any future agent that understands the format.

## Architecture

```
RH repo
├── skills/                   single source of truth
│   ├── manifest.json         machine-readable index (regenerated by `rh skill index`)
│   ├── paper-writing/        ← one folder per skill
│   │   ├── SKILL.md
│   │   └── tracks/...
│   └── agent-profiles/       templates for `.rh-agent.toml`
│       ├── claude-code.toml
│       ├── codex.toml
│       └── openclaw.toml
└── packages/research_harness/.../skill/   CLI + library
```

Three layers, each independent:

1. **Source** — the `skills/` directory plus its generated `manifest.json`.
   No agent-specific assumptions.
2. **Agent self-declaration** — every consuming agent ships its own
   `.rh-agent.toml` declaring `install_path`, `format`, and `strategy`. RH
   reads this file and obeys it. RH does not maintain a list of supported
   agents.
3. **Delivery** — three strategies are supported:
    - `symlink` (default): each skill folder is symlinked into the agent's
      install path. Updates flow automatically.
    - `copy`: a recursive copy. Used when the agent's loader doesn't follow
      symlinks, or in CI / sandboxes.
    - `mcp-runtime`: nothing is written to disk; the agent calls the MCP
      tools `skill_list` and `skill_get` at runtime instead.

## Quick start

```bash
# 1. Build the manifest (setup.sh does this automatically).
rh skill index

# 2. Install for whichever agent you're using.
rh skill install --agent claude-code      # uses skills/agent-profiles/claude-code.toml
rh skill install --target ~/.claude/skills  # ad-hoc, no profile
rh skill install                           # auto: looks for ./.rh-agent.toml

# 3. Verify what's installed.
rh skill verify --agent claude-code
```
