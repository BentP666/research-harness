# Track E: 立场论文 (Position Paper)

提出一个观点/主张/视角，用论证（而非实验）说服读者。目标 venue: TMLR, HotNets, 部分 workshop 的 position track。

## 核心叙事

立场论文不是展示实验结果，而是**提出一个有根据的、有争议性的主张**：

```
1. 学术界/产业界普遍认为 [conventional wisdom]
2. 然而这一共识存在 [根本性的问题/盲区]
3. 我们主张 [contrarian/nuanced position]
4. 支撑论据: [理论分析/案例研究/思想实验/数据分析]
5. 如果我们的主张成立，implications 是 [对研究方向/实践的影响]
```

## Section Blueprint

| Section | 核心任务 | 关键检查 |
|---------|----------|----------|
| Abstract | 主张 + 为什么重要 + 核心论据 | 立场是否清晰、有争议性？ |
| Introduction | 挑战 conventional wisdom + 提出主张 | 读者会不会觉得"有意思，但我不同意"？ |
| Background | 现有共识是什么 + 为什么形成 | 公平呈现对方观点了吗？ |
| Argument | 逐层论证（2-3 个支撑论据） | 每个论据独立且有说服力吗？ |
| Discussion | Implications + limitations + how to test | 承认了自己主张的局限性吗？ |
| Conclusion | Restate + call to action | 明确了读者应该做什么不同？ |

## 导师原则适配

### P3-P8（六维验证）→ 为什么这个观点现在重要

立场论文的问题定位不是"问题重要"而是"共识有误"：

- 这个 conventional wisdom 有多广泛？（影响多少研究？）
- 如果共识有误，后果多严重？（误导多少资源？）
- 为什么现在提出这个观点？（什么新证据/趋势使重新思考成为必要？）

### P9-P10 → 2-3 个论据 → 2-3 个 implications

```
Arguments:
  ❶ 论据1: [理论层面的论证]
  ❷ 论据2: [实证层面的证据]
  ❸ 论据3: [案例/反例分析]

Implications:
  ❶ 如果论据1成立 → 研究方向应该 [调整]
  ❷ 如果论据2成立 → 评估方法应该 [改变]
  ❸ 如果论据3成立 → 实践中应该 [注意]
```

### P11（包装）→ Provocative but defensible

- 主张要够大胆，让人想反驳（provocative）
- 但论据要扎实，反驳起来不容易（defensible）
- 明确说出"如果我们错了，代价很低；如果我们对了，impact 很大"

## 写作要点

- **不要伪装成方法论文**——立场论文的价值在观点，不在实验
- **公平呈现对方观点**——strawman 式的批评会被审稿人立刻识破
- **具体化主张**——"X doesn't work" 太弱，"X fails under condition Y because Z" 有力
- **提供 testable predictions**——让读者能验证/反驳你的主张

## 质量检查

- [ ] 主张在 abstract 第一句清晰表达
- [ ] 公平呈现了 conventional wisdom
- [ ] 每个论据独立有力（不是同一论据的变形）
- [ ] 明确了 implications 和 how to test
- [ ] 承认了主张的局限性（不过度宣称）
