# Track D: 短论文 / Workshop Paper

4-6 页，快速发表初步成果或有趣 insight。目标 venue: Workshop papers, Findings (ACL), Short papers (EMNLP), Demo papers。

## 核心叙事

短论文的叙事极度精炼——**一个 insight + 初步证据**：

```
1. [一个领域的具体问题]
2. 我们发现/提出 [一个具体的 insight/方法]
3. 初步实验显示 [有前景的结果]
4. 这为 [更大的研究方向] 提供了基础
```

## Section Blueprint

| Section | 篇幅 | 核心任务 |
|---------|------|----------|
| Abstract | 100 words | 问题 + insight + 关键数字 |
| Introduction | 0.5-0.75 页 | 问题 + 方法 + 1-2 条贡献 |
| Method | 1-1.5 页 | 精炼描述，细节放 appendix |
| Experiments | 1-1.5 页 | 最核心的 1-2 个实验 |
| Related Work | 0.5 页 | 精简定位 |
| Conclusion | 2-3 句 | 总结 + future work |

## 导师原则适配

### P9-P10 → 1-2 个问题 → 1-2 个贡献

短论文不需要 3→3 对称，1-2 条足矣：

```
Gap: [一个具体的、聚焦的问题]
Contribution: [一个直接对应的 insight/方法/发现]
```

### P1（引用分级）→ 极度精选

- 15-25 篇总引用
- 几乎全部 ★★★
- 只引与你的 insight 直接相关的工作

### P11（包装）→ 突出 provocativeness

短论文不需要证明"我解决了问题"，需要证明"这个方向有价值"：
- 聚焦于 surprising finding
- 展示初步但有说服力的证据
- 明确指出 future work 方向

## 写作要点

- **每个句子都要有信息量**——短论文没有空间容纳铺垫和过渡
- **压缩到极致后再压缩一轮**——目标是在 4 页内说清楚核心
- **实验细节放 appendix**——主文只放最关键的结果
- **Figure 1 即核心**——在短论文中，Figure 1 几乎 = 整个方法

## 质量检查

- [ ] 核心 insight 能用一句话说清楚
- [ ] 主文不超过 page limit
- [ ] 没有"未来我们会做X"来弥补缺失的实验
- [ ] 引用全部 ★★★ 或有充分理由的 ★★
